前言

单例模式（Singleton Pattern）是设计模式中最简单、最常用的一种。它确保一个类只有一个实例，并提供一个全局访问点。

虽然看似简单，但在多线程环境下，单例模式的实现却暗藏玄机。本文将详细介绍 7 种单例模式的实现方式，从最基本的饿汉式到最优雅的枚举式，并分析各自的优缺点和适用场景。

一、什么是单例模式

1.1 定义

单例模式确保一个类只有一个实例，并提供一个全局访问点来获取该实例。

1.2 使用场景

  • 配置管理类（如读取配置文件）
  • 数据库连接池
  • 线程池
  • 缓存（如 Redis 客户端）
  • 日志记录器
  • 应用状态管理器

1.3 实现要点

  1. 私有化构造方法：防止外部通过 new 创建实例
  2. 持有唯一实例：在类内部创建并持有实例
  3. 提供全局访问方法：通过静态方法获取实例

2.3 优缺点

优点：

  • 实现简单，无需考虑线程安全问题
  • 没有加锁，性能最好
  • 天然防止反射和序列化破坏（配合 final）

缺点：

  • 不支持延迟加载，可能造成资源浪费
  • 如果实例创建耗时或占用资源大，会影响类加载速度

2.4 适用场景

  • 实例占用资源小、创建速度快
  • 确定会被使用到的单例
  • 对启动速度不敏感的场景

四、静态内部类（Holder 模式，推荐）

4.1 原理

  • JVM 保证类的初始化是线程安全的（类加载器机制）
  • 静态内部类 Holder 只有在被首次引用时才会加载
  • 实现了延迟加载，且无需加锁

4.2 特点分析

实现复杂度⭐⭐ 较简单
性能⭐⭐⭐⭐⭐ 无锁，性能最好

4.3 优点

  • 既实现了延迟加载，又保证了线程安全
  • 没有性能损耗
  • 代码简洁优雅

五、枚举式（Enum，最推荐）

5.1 为什么是最佳实践？

Joshua Bloch（《Effective Java》作者）推荐的方式，原因：

  3. 防止序列化破坏：枚举反序列化时返回相同的实例
  4. 代码简洁：一行代码实现单例

六、容器式（注册表模式）

6.1 适用场景

  • 需要管理多个单例实例
  • 需要动态注册/获取单例
  • 框架级单例管理（如 Spring 的 Bean 容器）

七、ThreadLocal 单例（线程隔离）

7.1 特点

  • 每个线程拥有独立的实例
  • 严格来说不是单例，而是 “线程单例”
  • 适用于需要线程隔离的场景（如数据库连接）

十一、面试题精选

Q1：单例模式有哪些实现方式？哪种最好？

答：常见的有 7 种实现方式：

  1. 饿汉式：类加载时创建，线程安全，不支持延迟加载
  2. 懒汉式（同步方法）：延迟加载，但性能差
  3. 双重检查锁定（DCL）：延迟加载 + 线程安全，需要 volatile
  4. 静态内部类：延迟加载 + 线程安全 + 高性能，最优雅的延迟加载方案
  5. 枚举式：最简单、最安全，天然防止反射和序列化攻击，Joshua Bloch 推荐
  6. 容器式：管理多个单例，适用于框架
  7. ThreadLocal：线程隔离的"单例"

推荐：

  • 不需要延迟加载：枚举式
  • 需要延迟加载：静态内部类

Q2：双重检查锁定为什么要用 volatile？

答：instance = new Singleton() 不是原子操作，可能重排序为：

  1. 分配内存空间
  2. 初始化对象
  3. 将引用指向内存地址

如果重排序为 1 → 3 → 2，线程 A 执行完 3 后，线程 B 检查 instance != null，会返回一个未初始化完成的对象。

volatile 禁止指令重排序，保证 2 → 3 的执行顺序。

Q3：静态内部类为什么能实现延迟加载？

答：利用了 JVM 的类加载机制：

  • 静态内部类 Holder 只有在被首次引用时才会加载
  • SingletonHolder.getInstance() 首次调用时，触发 Holder 类加载
  • 类加载时初始化 INSTANCE，JVM 保证类初始化的线程安全

这样既实现了延迟加载，又无需加锁，性能最优。

Q4：枚举单例是如何防止反射和序列化攻击的？

答：

序列化：枚举反序列化时，通过 name 在枚举常量中查找已存在的实例，不会创建新对象。

Q5：单例模式的优缺点是什么？

答：

优点：

  • 内存中只有一个实例，节省资源
  • 全局访问点，方便使用
  • 避免重复创建，提高性能

缺点：

  • 违反单一职责原则（类既负责业务逻辑又负责创建管理）
  • 隐藏了类之间的依赖关系
  • 不利于单元测试（难以 mock）
  • 扩展困难（继承后不再是单例）

Q6：Spring 中的单例 Bean 是如何实现的？

答：Spring 使用 容器式单例（注册表模式）：

  • DefaultSingletonBeanRegistry 使用 ConcurrentHashMap 存储单例 Bean

  • 值：Bean 实例

Spring 的单例作用域是 Spring 容器级别，不是 JVM 级别。

十二、总结

单例模式虽然简单，但在多线程环境下实现却有很多细节需要注意：

  1. 饿汉式：简单高效，但不支持延迟加载
  2. DCL：支持延迟加载，需要 volatile 防止指令重排序
  3. 静态内部类：延迟加载的最佳实践，无锁高性能
  4. 枚举式：最安全的实现，天然防止反射和序列化攻击

实际开发建议：

  • 优先使用 枚举式（Effective Java 推荐）
  • 需要延迟加载时用 静态内部类
  • 避免使用同步方法版的懒汉式（性能差）
  • 注意防御反射、序列化、克隆攻击
