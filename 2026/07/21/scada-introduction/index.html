一、写在前面

最近面试被问到"你会 SCADA 吗？"，我第一反应是：不会，完全没接触过。

但回头一想，我其实用过 PLC（西门子），用过 KepServer（OPC 服务器），还调研过 JetLinks（物联网平台）。这些东西凑在一起，不就是 SCADA 的各个零件吗？

本文是我自己对 SCADA 的学习整理，从一个"用过零件但没见过整车"的视角出发，把 SCADA 讲清楚。

二、一句话理解 SCADA

SCADA = Supervisory Control And Data Acquisition，监控与数据采集系统。

如果用人话翻译：

SCADA 就是工厂的"大屏监控系统"——把现场所有 PLC、传感器、仪表的数据采集上来，画成曲线图、表格、报警灯，让操作员在一台电脑上就能看到全场情况，还能远程控制设备。

三、你已有的知识，在 SCADA 中是什么角色

你熟悉的在 SCADA 中的角色说明
PLC（西门子）现场控制层直接控制电机、阀门、传感器，是 SCADA 的"手脚"
KepServer通信网关 / OPC Server把不同品牌 PLC 的私有协议转成统一标准（OPC），给上层用
JetLinks物联网平台层设备接入、数据存储、规则引擎，类似 SCADA 的后端

把它们串起来：

PLC（西门子） ──私有协议──► KepServer ──OPC──► SCADA 上位机 │ └──MQTT──► JetLinks（云平台）

你会发现：SCADA 不是某一个产品，而是一套"采集→传输→展示→控制"的完整系统架构。

四、SCADA 的系统架构

一个典型的 SCADA 系统分为四层：

┌────────────────────────────────────────────────┐ │ 第 4 层：管理层（Management） │ │ ERP、MES、企业管理系统、云平台（JetLinks） │ │ 做数据统计、报表、生产调度 │ └──────────────────────┬─────────────────────────┘ │ HTTP / MQTT / 数据库同步 ┌──────────────────────▼─────────────────────────┐ │ 第 3 层：监控层（SCADA 主机） │ │ WinCC / IFIX / 组态王 / Ignition / 力控 │ │ 实时画面、曲线、报警、操作、历史数据 │ └──────────────────────┬─────────────────────────┘ │ OPC / Modbus / 私有协议 ┌──────────────────────▼─────────────────────────┐ │ 第 2 层：通信层（网关 / OPC Server） │ │ KepServer / OPC Router / 工业网关 │ │ 协议转换：西门子S7 → OPC → Modbus TCP │ └──────────────────────┬─────────────────────────┘ │ Profinet / Profibus / RS485 ┌──────────────────────▼─────────────────────────┐ │ 第 1 层：现场层（设备层） │ │ PLC、RTU、传感器、变频器、仪表、阀门 │ │ 直接与物理世界交互 │ └────────────────────────────────────────────────┘

各层对应你熟悉的东西

层级你熟悉的你还不熟的
第 4 层JetLinksERP / MES
第 3 层❌ 完全空白WinCC / IFIX / 组态王
第 2 层KepServer ✅工业网关
第 1 层西门子 PLC ✅RTU / 变频器

所以面试官问的"你会 SCADA 吗"，大概率问的是第 3 层——那些组态软件。

五、主流 SCADA 软件盘点

软件厂商特点适用场景
WinCC西门子和西门子 PLC 深度绑定，功能强大，价格贵西门子生态
IFIXGE老牌，稳定性好，电力行业多电力、石化
组态王亚控国产，性价比高，操作简单中小型项目
力控力控科技国产，电力行业多电力、能源
IgnitionInductive Automation跨平台、Web 技术、无限授权现代化工厂
IntouchAVEVA老牌，市场占有率高大型项目

选型建议

  • 如果甲方全是西门子设备 → WinCC（生态绑定，省心）
  • 如果预算有限、项目中小型 → 组态王 / 力控（国产够用）
  • 如果想用现代 Web 技术 → Ignition（Java 技术栈，对你友好）
  • 如果甲方是电力/石化 → 跟着甲方走，IFIX 或力控

六、SCADA 的核心功能

6.1 实时监控（HMI - 人机界面）

把工厂画成一张图，用动画展示设备状态：

┌─────────────────────────────────────────┐ │ ┌─────┐ ┌─────┐ ┌─────┐ │ │ │ 泵1 │ │ 泵2 │ │ 泵3 │ │ │ │ 🟢运行│ │ 🔴停止│ │ 🟡故障│ │ │ └─────┘ └─────┘ └─────┘ │ │ │ │ 温度: 23.5°C 压力: 1.2 MPa │ │ 流量: 156 m³/h 液位: 2.8 m │ └─────────────────────────────────────────┘

6.2 报警管理

  • 温度超限 → 弹窗 + 声音 + 短信通知
  • 设备离线 → 红色闪烁
  • 报警分级：紧急 / 重要 / 一般
  • 报警确认：必须人工确认才能消除

6.3 趋势曲线

  • 实时曲线：过去 1 小时，刷新频率 1 秒
  • 历史曲线：过去 3 个月，用于分析设备劣化趋势

6.4 数据存储

  • 关键数据每秒存一次
  • 历史数据存关系库（SQL Server / MySQL）或时序库（InfluxDB / TDengine）
  • 报表导出：日报、月报、年报

6.5 远程控制

  • 点击按钮远程启停设备
  • 设定 PID 参数
  • 操作记录审计：谁、几点、干了什么

七、SCADA 和 JetLinks 的关系

你调研过 JetLinks，这里对比一下：

维度SCADA 上位机JetLinks
定位工业现场监控物联网设备管理平台
部署位置工厂内部服务器云端 / 私有化部署
界面组态画面（流程图风格）Web Dashboard
协议OPC / Modbus / ProfibusMQTT / CoAP / HTTP
实时性毫秒级秒级
数据量几千个点几十万设备
控制直接控制设备通过规则引擎下发指令

简单说：SCADA 管一个工厂，JetLinks 管一堆设备。 两者可以互补——SCADA 负责现场实时控制，JetLinks 负责云端数据汇聚和分析。

八、如何从零开始学 SCADA

第一步：理解组态软件

下载一个组态王试用版，熟悉：

  • 变量定义（把 PLC 的地址映射成变量）
  • 画面组态（拖控件、绑定变量、做动画）
  • 报警配置（设置阈值、触发条件）
  • 报表模板

第二步：把自己已有的知识串起来

西门子 PLC ──S7协议──► KepServer ──OPC──► 组态王 │ └──MQTT──► JetLinks

你少的就是"组态王"这一个环节。 只要把 OPC 的数据接到组态王，你就能搭出一个完整的 SCADA 系统。

第三步：了解常用协议

协议用途你掌握程度
Modbus RTU/TCP通用设备协议已会 ✅
S7（西门子私有）西门子 PLC通过 KepServer 间接用
OPC UA工业互联标准KepServer 就是 OPC Server
MQTT物联网协议已会 ✅
Profibus / Profinet西门子现场总线了解即可

第四步：实践项目

  • 用 KepServer 模拟一个 PLC 设备
  • 用组态王连接 KepServer 的 OPC 接口
  • 画一个简单的工艺流程图，绑定变量
  • 配置报警和趋势曲线

九、面试时怎么说

如果面试官问"你会 SCADA 吗？"，你可以这样回答：

“我目前主要做的是 SCADA 的底层和通信层——用过西门子 PLC、KepServer 做协议转换，也调研过 JetLinks 做设备管理平台。SCADA 上层的组态软件（比如 WinCC、组态王）我没有直接用过，但我理解整个数据流：PLC → KepServer → OPC → 组态软件，这块如果再学一下组态软件，可以很快上手。”

这样回答，既诚实，又展示了你的知识体系。

十、总结

你想知道的答案
SCADA 是什么工厂的"大屏监控系统"，采集 + 展示 + 报警 + 控制
你缺什么组态软件（WinCC / 组态王 / 力控等）
你已有的PLC、KepServer、JetLinks 都是 SCADA 的零件
能不能学完全能，你只差最后一层组态软件的配置
学多久1 周摸熟操作，1 个月能独立做项目

面试题精选

1. SCADA 的全称是什么？核心功能有哪些？

SCADA = Supervisory Control And Data Acquisition（监控与数据采集系统）。

核心功能包括：实时监控（HMI）、报警管理、趋势曲线、数据存储、远程控制、报表生成。

2. SCADA 系统分为哪几层？

层级名称内容
第 4 层管理层ERP、MES
第 3 层监控层SCADA 上位机（WinCC、组态王等）
第 2 层通信层OPC Server、工业网关（KepServer）
第 1 层现场层PLC、RTU、传感器、仪表

3. OPC 是什么？为什么需要它？

OPC（OLE for Process Control）是工业通信的标准接口。不同品牌的 PLC 有各自的私有协议（西门子 S7、三菱 MC、欧姆龙 FINS 等），如果每个上位机都去适配，工作量巨大。

有了 OPC Server（如 KepServer），它负责把各种私有协议统一转成 OPC 标准接口，上位机只需要对接 OPC 即可，一处对接，通用所有设备。

4. KepServer 和 SCADA 上位机有什么区别？

KepServer 是通信中间件，只负责协议转换和数据转发，不提供画面展示。SCADA 上位机（如 WinCC）是完整的人机界面，负责画面展示、报警、曲线、报表等。

两者关系：KepServer 采集数据 → 通过 OPC 给 SCADA 上位机 → 上位机展示。

5. SCADA 和 DCS 有什么区别？

维度SCADADCS
控制范围地域分散（输油管道、水厂）地域集中（化工厂装置）
实时性要求较低（秒级）要求高（毫秒级）
控制方式以监控为主，少量控制以控制为主，大量闭环调节
典型场景水处理、电力调度、油气管线炼油、化工、制药

6. SCADA 和物联网平台（如 JetLinks）有什么区别？

维度SCADA物联网平台
定位工业现场实时监控海量设备远程管理
实时性毫秒~秒级秒级即可
协议OPC / Modbus / ProfibusMQTT / CoAP / HTTP
部署工厂本地云端
设备量几百~几千点几十万+

两者可以互补：SCADA 负责现场实时控制，物联网平台负责云端数据汇聚。
