一、HashMap 概述

HashMap 是 Java 中最常用的数据结构之一，它基于哈希表实现，提供了 O(1) 平均时间复杂度的增删改查操作。从 JDK 1.8 开始，HashMap 引入了红黑树优化，显著提升了哈希冲突严重时的性能。

核心特点

  • 线程不安全：多线程环境下需使用 ConcurrentHashMap
  • 允许 null 键和 null 值
  • 不保证顺序：遍历顺序可能与插入顺序不同
  • 动态扩容：负载因子默认 0.75，达到阈值自动扩容

4.3 红黑树特性

红黑树是一种自平衡二叉搜索树，满足以下规则：

  1. 每个节点非红即黑
  2. 根节点是黑色
  3. 红色节点的子节点必须是黑色（不能有连续红色节点）
  4. 从任一节点到其每个叶子的所有路径包含相同数量的黑色节点
  5. 叶子节点（NIL）是黑色

九、总结

核心要点

  1. 数据结构演进：JDK 1.8 引入红黑树，解决了链表过长时的性能问题
  2. 链表转红黑树：需要同时满足链表长度 ≥ 8 和数组长度 ≥ 64
  3. 尾插法：避免了扩容时的死循环问题
  4. 阈值设计：8 和 6 的差值防止频繁转换

面试高频问题

问题答案要点
HashMap 为什么线程不安全？扩容时数据丢失、死循环（1.7）、数据覆盖
为什么用红黑树不用 AVL？红黑树插入删除效率更高，AVL 查询略快但维护成本高
负载因子为什么是 0.75？时间和空间的平衡，泊松分布计算得出
HashMap 的扩容机制？2 倍扩容，重新计算索引，保持顺序
