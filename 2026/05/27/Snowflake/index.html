雪花算法是推特公司开源的一种分布式ID生成算法，用于在高并发场景下生成全局唯一的ID。

雪花算法结构：
1bit（符号位）+ 41bit（时间戳）+ 10bit（机器ID）+ 12bit（序列号）

雪花算法的局限性:

  1. 强依赖机器时钟：时钟回拨会导致 ID 重复 (最需要解决的问题)
  2. ID 不是完全随机，会暴露数据量、业务规律等信息 (低)
  3. 单机房单实例每秒并发有限制：4096 个ID (低)
  4. 机器 ID（WorkerID）分配与管理麻烦 (低)
  5. 时间位数固定，能用的年限有限 (低)
  6. 分布式多机房部署容易冲突 (低)

解决方法大致三种一种是美团的leaf, 百度的UidGenerator, 和滴滴的Tinyid
