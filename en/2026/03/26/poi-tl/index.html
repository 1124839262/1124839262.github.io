Poi-tl Documentation

Excerpt

Word template engine

Plugins, also known as custom functions, allow users to execute pre-defined functions at template tag positions. Thanks to the plugin mechanism, we can perform virtually any operation at any position in the template.

Plugins are the core mechanism of poi-tl. All data rendering in this documentation is implemented through built-in plugins.

Basic Usage

Text Tag {{var}}

Data model: String

Image Tag {{@var}}

Data model: PictureRenderData

Table Tag {{#var}}

Data model: TableRenderData

Numbering Tag {{*var}}

Data model: NumberingRenderData

Document Tag {{+var}}

Data model: DocxRenderData

Advanced Features

Dynamic Table

Use TableRenderPolicy to dynamically create table rows:

Comment Plugin

CommentRenderPolicy is a built-in plugin that provides full comment functionality support.

Data model: CommentRenderData

Comments support adding text, images, and other document content.

Attachment Plugin

AttachmentRenderPolicy is a built-in plugin that supports inserting attachments.

Data model: AttachmentRenderData

Inserts an Excel file into the document; double-click the icon to open the attachment.

Code Highlighting

HighlightRenderPolicy plugin provides syntax highlighting for code blocks in Word documents.

Dependency:

Data model: HighlightRenderData

Supports 20+ programming languages and dozens of theme styles.

Common language support: apache, bash, cpp, cs, css, diff, go, groovy, http, ini, java, javascript, json, makefile, markdown, objectivec, perl, php, python, ruby, scala, shell, sql, xml, yaml

Common theme styles: github, idea, zenburn, androidstudio, solarized-light, solarized-dark, xcode, vs, agate, darcula, dark, dracula, foundation, googlecode, monokai, mono-blue, far, gml

Markdown Plugin

MarkdownRenderPolicy plugin supports generating Word documents from Markdown.

Dependency:

Data model: MarkdownRenderData

Converts Markdown content to Word documents.
