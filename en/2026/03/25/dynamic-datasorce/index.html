Dynamic Datasource Documentation

dynamic-datasource-spring-boot-starter is a starter for quickly integrating multiple data sources based on Spring Boot.

Here is the organized documentation.

Core Concepts

  • @DS: Annotation for switching data sources, can be placed on methods or classes
  • @DSTransactional: Local transaction annotation for multi-datasource scenarios
  • DynamicDataSourceContextHolder: Programmatic datasource switching

Important Notes

  • @DSTransactional uses local transactions, not true distributed transactions
  • For true distributed transactions, use Seata AT mode
  • @Transactional and @DSTransactional should not be mixed

Common Issues

Can @Transactional and @DS be used together?

Yes. @DS switches the datasource first, then @Transactional opens the transaction. Within the entire native transaction, no further datasource switching is possible - it will always use the initial datasource.

So if you confirm that no other datasource will be switched within the entire transaction, using native @Transactional is recommended as it is more complete.

What does “local transactions and Spring native transactions should not be mixed” mean?

Scenario 1: Using Spring @Transactional first, then calling a method with @DSTransactional inside.

Based on the core principle: under a transaction, all operations use the same datasource. Nothing inside can change that.

Scenario 2: Using @DSTransactional first, then calling a method with @Transactional inside.

B and C are independent transactions. If C fails, will B roll back? No, B has already committed. Will A roll back? Yes.

Mixing is not recommended unless you fully understand transaction behavior and can control your code execution flow.
