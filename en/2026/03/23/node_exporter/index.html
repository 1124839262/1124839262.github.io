❓ Common Issues

  • Port Conflict: If port 9100 is occupied, modify the systemd file with ExecStart=/usr/local/bin/node_exporter --web.listen-address=":9101" and update the port in Prometheus configuration.

Installation on Windows

Installing windows_exporter (formerly wmi_exporter) on Windows is also simple—it’s a standalone binary with no complex dependencies.

Below are standard production environment deployment steps (including creating a service and auto-start on boot).

Installing windows_exporter (formerly wmi_exporter) on Windows is straightforward. It’s the official tool for collecting Windows system metrics (CPU, memory, disk, network, etc.).

Below are standard production installation steps, including download, service installation, firewall configuration, and verification.

🚀 Method 1: Using MSI Installer (Most Recommended, Simple and Fast)

This is the easiest method—the installer automatically handles service registration and firewall rules.

1. Download

Visit GitHub Releases to download the latest .msi installer.

  • Choose a file like windows_exporter-0.26.0-amd64.msi (based on your system architecture, usually amd64).

2. Install

  1. Double-click the downloaded .msi file.
  2. Click Next through the wizard.
  3. Key Step: In the “Custom Setup” or “Features” screen, select collectors to enable.
    • Default selection is usually fine.
    • For low-performance machines, uncheck unnecessary collectors (e.g., exchange, sql) and keep basic ones like cpu, memory, disk, net, os, service.
  4. Continue clicking Next until Install.
  5. The service will start automatically after installation.

3. Verify

Open a browser and visit:

If you see大量 text data, installation is successful!

💡 Common Parameter Reference

To customize behavior, add parameters during service installation:

  • --web.listen-address=":9182": Change listening port.
  • --collectors.enabled="cpu,memory,disk": Enable only specified collectors (improve performance).
  • --collectors.disabled="sql,exchange": Disable specific collectors.
  • --log.level="debug": Enable debug logs.
