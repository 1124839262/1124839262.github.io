Snowflake Algorithm

The Snowflake algorithm is a distributed ID generation algorithm developed by Twitter that generates 64-bit long IDs.

Structure

  • 1 bit: Sign bit, always 0
  • 41 bits: Timestamp (milliseconds), supports ~69 years
  • 10 bits: Machine ID (0-1023), supports 1024 machines
  • 12 bits: Sequence number (0-4095), supports 4096 IDs per millisecond per machine

Snowflake Issues

  1. Clock drift problem: When the system clock goes backward, duplicate IDs may be generated (High)
  2. Machine ID allocation: In containerized environments, auto-scaling makes machine ID management difficult (Medium)
  3. Sequence overflow: Under high concurrency, 4096 IDs per millisecond may not be enough (Low)
  4. Machine ID (WorkerID) allocation and management is troublesome (Low)
  5. Fixed timestamp bits limit the usable lifespan (Low)
  6. Distributed multi-datacenter deployment prone to conflicts (Low)

Solutions generally fall into three categories: Meituan’s Leaf, Baidu’s UidGenerator, and Didi’s TinyId.

We adopt Baidu’s UidGenerator combined with segment mode.
