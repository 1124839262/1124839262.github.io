Binary Search

Binary search, also known as half-interval search, is a highly efficient search algorithm that works on sorted arrays.

In a sorted array, it eliminates half of the invalid range each time to quickly locate the target value.

  • Time complexity: O(log n), far outperforming linear traversal for large datasets
  • Prerequisite: The array must be pre-sorted in ascending or descending order

Basic Search Principle

  1. Set left pointer and right pointer
  2. Take the middle position mid each time, compare the middle value with the target
  3. If middle value is too small -> target is in the right half, move left pointer right
  4. If middle value is too large -> target is in the left half, move right pointer left
  5. If equal, found; if range is exhausted without match, the target doesn’t exist

Standard Implementation

Bit Operation Binary Search

Bit operations are the most efficient
Only supports ascending sorted arrays
Can only return any matching index; cannot locate first/last position of duplicate elements

Recursive Binary Search

Prefer while iteration in production code

Left Boundary Binary Search

Array has duplicate values, returns the first index equal to the target

Right Boundary Binary Search

Array has duplicate values, returns the last index equal to the target

The core logic of all implementations is identical; only the range shrinking rules differ slightly.
