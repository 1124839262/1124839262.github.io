1. Introduction

As a Java backend developer, when you receive a requirement to “integrate with field devices,” your first reaction is often confusion — Java excels at web backends, but can it handle hardware like serial ports, PLCs, and sensors?

The answer is: absolutely. The Java ecosystem has rich communication libraries, and combined with Spring Boot, you can easily build a device management platform. This article introduces four mainstream approaches for Java to integrate with electrical devices from a practical development perspective.

3. Serial Communication (RS232/RS485)

3.1 Dependency

xml <dependency> <groupId>com.fazecast</groupId> <artifactId>jSerialComm</artifactId> <version>2.10.4</version> </dependency>

3.2 Scanning Available Serial Ports

`java
import com.fazecast.jSerialComm.SerialPort;

public class SerialScanner {
public static void main(String[] args) {
SerialPort[] ports = SerialPort.getCommPorts();
System.out.println(“Available serial ports:”);
for (SerialPort port : ports) {
System.out.printf(" %s - %s%n",
port.getSystemPortName(),
port.getDescriptivePortName());
}
}
}
`

Sample output:

Available serial ports: COM3 - USB-SERIAL CH340 (COM3) COM4 - Prolific USB-to-Serial (COM4)

3.3 Basic Send and Receive Example

`java
import com.fazecast.jSerialComm.SerialPort;

public class SerialDemo {
public static void main(String[] args) throws Exception {
SerialPort port = SerialPort.getCommPort(“COM3”);
port.setBaudRate(9600); // Baud rate
port.setNumDataBits(8); // Data bits
port.setNumStopBits(1); // Stop bits
port.setParity(SerialPort.NO_PARITY); // Parity

    port.openPort();
    System.out.println("Serial port opened: " + port.isOpen());

    // Send data
    byte[] sendData = {0x01, 0x03, 0x00, 0x00, 0x00, 0x01, (byte) 0x84, 0x0A};
    port.writeBytes(sendData, sendData.length);
    System.out.println("Sent: " + bytesToHex(sendData));

    // Read response
    Thread.sleep(200);
    byte[] buffer = new byte[256];
    int len = port.readBytes(buffer, buffer.length);
    byte[] response = new byte[len];
    System.arraycopy(buffer, 0, response, 0, len);
    System.out.println("Response: " + bytesToHex(response));

    port.closePort();
}

private static String bytesToHex(byte[] bytes) {
    StringBuilder sb = new StringBuilder();
    for (byte b : bytes) {
        sb.append(String.format("%02X ", b));
    }
    return sb.toString().trim();
}

}
`

3.4 Important Notes

  • Baud rate, data bits, stop bits, and parity must match the device settings exactly, otherwise you will receive garbled data
  • Serial ports are exclusive resources — only one program can open a port at a time
  • In production, consider using a connection pool to manage serial ports and avoid frequent opening/closing
  • RS485 is half-duplex communication — sending and receiving cannot happen simultaneously; you need to manage send/receive switching

4. Modbus TCP Communication

Modbus is the most widely used communication protocol in industrial automation. Modbus TCP runs Modbus over Ethernet, making Java integration very convenient.

4.1 Dependency

xml <dependency> <groupId>com.ghgande</groupId> <artifactId>j2mod</artifactId> <version>3.2.0</version> </dependency>

4.2 Reading Holding Registers

`java
import com.ghgande.j2mod.modbus.Modbus;
import com.ghgande.j2mod.modbus.io.ModbusTCPTransaction;
import com.ghgande.j2mod.modbus.msg.ReadMultipleRegistersRequest;
import com.ghgande.j2mod.modbus.msg.ReadMultipleRegistersResponse;
import com.ghgande.j2mod.modbus.net.TCPMasterConnection;

import java.net.InetAddress;

public class ModbusReader {
public static void main(String[] args) throws Exception {
// Connect to device (Modbus default port 502)
InetAddress addr = InetAddress.getByName(“192.168.1.100”);
TCPMasterConnection conn = new TCPMasterConnection(addr);
conn.setPort(502);
conn.connect();
System.out.println(“Modbus TCP connected”);

    // Read holding registers 40001~40002 (address starts at 0, so read 0~1)
    ReadMultipleRegistersRequest request = new ReadMultipleRegistersRequest(0, 2);
    request.setUnitID(1); // Slave address

    ModbusTCPTransaction transaction = new ModbusTCPTransaction(conn);
    transaction.setRequest(request);
    transaction.execute();

    ReadMultipleRegistersResponse response =
        (ReadMultipleRegistersResponse) transaction.getResponse();
    System.out.println("Register 40001 value: " + response.getRegisterValue(0));
    System.out.println("Register 40002 value: " + response.getRegisterValue(1));

    conn.close();
}

}
`

4.3 Writing a Single Register

`java
import com.ghgande.j2mod.modbus.msg.WriteSingleRegisterRequest;
import com.ghgande.j2mod.modbus.msg.WriteSingleRegisterResponse;

// Write value 1000 to register 40001
WriteSingleRegisterRequest writeReq = new WriteSingleRegisterRequest(0, 1000);
writeReq.setUnitID(1);

ModbusTCPTransaction writeTrans = new ModbusTCPTransaction(conn);
writeTrans.setRequest(writeReq);
writeTrans.execute();

WriteSingleRegisterResponse writeResp =
(WriteSingleRegisterResponse) writeTrans.getResponse();
System.out.println("Write result: " + writeResp.getRegisterValue());
`

5. MQTT IoT Protocol

MQTT is a lightweight publish/subscribe messaging protocol ideal for device data reporting to cloud platforms.

5.1 Dependency

xml <dependency> <groupId>org.eclipse.paho</groupId> <artifactId>org.eclipse.paho.client.mqttv3</artifactId> <version>1.2.5</version> </dependency>

5.2 Subscribing to Device Data

`java
import org.eclipse.paho.client.mqttv3.*;
import org.eclipse.paho.client.mqttv3.persist.MemoryPersistence;

public class MqttSubscriber {
public static void main(String[] args) throws Exception {
String broker = “tcp://192.168.1.10:1883”;
String clientId = “java-backend-” + System.currentTimeMillis();
String topic = “factory/workshop1/temperature”;

    MqttClient client = new MqttClient(broker, clientId, new MemoryPersistence());

    MqttConnectOptions options = new MqttConnectOptions();
    options.setCleanSession(true);
    options.setUserName("admin");
    options.setPassword("password".toCharArray());

    client.setCallback(new MqttCallback() {
        @Override
        public void connectionLost(Throwable cause) {
            System.out.println("MQTT connection lost: " + cause.getMessage());
        }

        @Override
        public void messageArrived(String topic, MqttMessage message) {
            String payload = new String(message.getPayload());
            System.out.printf("Data received [%s]: %s%n", topic, payload);
            // Store to database, trigger alerts, etc.
        }

        @Override
        public void deliveryComplete(IMqttDeliveryToken token) {
        }
    });

    client.connect(options);
    client.subscribe(topic, 1); // QoS 1
    System.out.println("Subscribed to: " + topic);

    // Keep running
    Thread.sleep(Long.MAX_VALUE);
}

}
`

5.3 Sending Control Commands

java // Publish control command to actuator String controlTopic = "factory/workshop1/valve/control"; String command = "{\"valve_id\":\"V001\",\"action\":\"open\",\"value\":80}"; MqttMessage msg = new MqttMessage(command.getBytes()); msg.setQos(1); msg.setRetained(false); client.publish(controlTopic, msg); System.out.println("Command sent: " + command);

6. OPC UA Integration

OPC UA is the core communication standard for Industry 4.0, supporting cross-platform, security authentication, and structured data.

6.1 Dependency

xml <dependency> <groupId>org.eclipse.milo</groupId> <artifactId>sdk-client</artifactId> <version>0.6.11</version> </dependency>

6.2 Reading Node Data

`java
import org.eclipse.milo.opcua.sdk.client.OpcUaClient;
import org.eclipse.milo.opcua.stack.core.types.builtin.NodeId;
import org.eclipse.milo.opcua.stack.core.types.builtin.DataValue;
import org.eclipse.milo.opcua.stack.core.types.enumerated.TimestampsToReturn;

import java.util.concurrent.CompletableFuture;

public class OpcUaReader {
public static void main(String[] args) throws Exception {
String endpoint = “opc.tcp://192.168.1.100:4840”;

    OpcUaClient client = OpcUaClient.create(endpoint);
    client.connect().get();
    System.out.println("OPC UA connected");

    // Read node (temperature sensor)
    NodeId nodeId = new NodeId(2, "TemperatureSensor.Value");
    DataValue value = client.readValue(
        0.0, TimestampsToReturn.Both, nodeId
    ).get();

    System.out.println("Temperature value: " + value.getValue().getValue());
    System.out.println("Timestamp: " + value.getSourceTime());

    client.disconnect().get();
}

}
`

6.3 Subscribing to Data Changes

java // Create subscription: push data actively when sensor value changes, no polling needed client.getSubscriptionManager().createSubscription(1000.0).thenAccept(sub -> { NodeId nodeId = new NodeId(2, "TemperatureSensor.Value"); sub.createMonitoredItem( new MonitoringParameters(1, 100.0, null, 0), new MonitoredItemCreateRequest(nodeId, AttributeId.Value.uid()) ).thenAccept(item -> { item.setValueConsumer((monitoredItem, value) -> { System.out.println("Temperature changed: " + value.getValue().getValue()); }); }); });

Core Code: Communication Adapter Interface

`java
public interface DeviceAdapter {
/** Connect to device */
boolean connect();

/** Disconnect */
void disconnect();

/** Read data */
Map<String, Object> read(List<String> points);

/** Write data */
boolean write(String point, Object value);

/** Check if connected */
boolean isConnected();

}
`

Modbus Adapter Implementation Example

`java
@Component
public class ModbusDeviceAdapter implements DeviceAdapter {

private TCPMasterConnection connection;
private final String host;
private final int port;
private final int unitId;

public ModbusDeviceAdapter(
        @Value("") String host,
        @Value("") int port,
        @Value("") int unitId) {
    this.host = host;
    this.port = port;
    this.unitId = unitId;
}

@Override
public boolean connect() {
    try {
        connection = new TCPMasterConnection(InetAddress.getByName(host));
        connection.setPort(port);
        connection.connect();
        return true;
    } catch (Exception e) {
        log.error("Modbus connection failed", e);
        return false;
    }
}

@Override
public Map<String, Object> read(List<String> points) {
    Map<String, Object> result = new HashMap<>();
    try {
        ReadMultipleRegistersRequest request =
            new ReadMultipleRegistersRequest(0, points.size());
        request.setUnitID(unitId);

        ModbusTCPTransaction transaction = new ModbusTCPTransaction(connection);
        transaction.setRequest(request);
        transaction.execute();

        ReadMultipleRegistersResponse response =
            (ReadMultipleRegistersResponse) transaction.getResponse();
        for (int i = 0; i < points.size(); i++) {
            result.put(points.get(i), response.getRegisterValue(i));
        }
    } catch (Exception e) {
        log.error("Failed to read registers", e);
    }
    return result;
}

@Override
public boolean write(String point, Object value) {
    try {
        int addr = Integer.parseInt(point);
        WriteSingleRegisterRequest request =
            new WriteSingleRegisterRequest(addr, ((Number) value).intValue());
        request.setUnitID(unitId);
        ModbusTCPTransaction transaction = new ModbusTCPTransaction(connection);
        transaction.setRequest(request);
        transaction.execute();
        return true;
    } catch (Exception e) {
        log.error("Failed to write register", e);
        return false;
    }
}

@Override
public void disconnect() {
    if (connection != null) {
        connection.close();
    }
}

@Override
public boolean isConnected() {
    return connection != null && connection.isConnected();
}

}
`

Scheduled Data Collection

`java
@Component
public class DataCollector {
@Autowired
private Listadapters;

@Scheduled(fixedDelay = 5000) // Collect every 5 seconds
public void collect() {
    for (DeviceAdapter adapter : adapters) {
        if (!adapter.isConnected()) {
            adapter.connect();
        }
        Map<String, Object> data = adapter.read(
            getDevicePoints(adapter));
        // Store in Redis for real-time frontend display
        redisTemplate.opsForHash().putAll(
            "device:data:" + adapter.getDeviceId(), data);
    }
}

}
`

8. Common Issues and Considerations

8.1 Connection Stability

  • Auto-reconnect: Must implement automatic reconnection using exponential backoff strategy
  • Heartbeat: Send periodic heartbeat packets to detect device online status
  • Connection Pool: Use connection pools for high-frequency collection scenarios

8.2 Data Reliability

  • QoS Level: Use QoS 1 for MQTT to ensure at-least-once delivery
  • Data Validation: Modbus has built-in CRC validation; consider adding custom checksums for serial communication
  • Data Backfill: After network interruption recovery, backfill data from the lost time period

8.3 Performance Optimization

  • Batch reading: Modbus supports reading multiple registers at once, reducing communication frequency
  • Async processing: Both OPC UA and MQTT support async callbacks, fully utilizing IO resources
  • Data compression: Compress high-frequency data before storing to database

8.4 Security Considerations

  • Do not use default passwords in production; Modbus itself has no authentication, use with firewall
  • MQTT must enable TLS encryption and username/password authentication
  • OPC UA supports certificate authentication — must be enabled in production

9. Summary

DimensionRecommended Solution
Direct PLC/SensorModbus TCP (most universal)
IoT Remote MonitoringMQTT + Spring Boot
Cross-platform Industrial ConnectivityOPC UA (Eclipse Milo)
Legacy Device SerialjSerialComm
Architecture DesignStrategy Pattern + Adapter Pattern, unified interface

Java integration with electrical devices is entirely feasible. The key is to choose the right library based on the device’s supported protocol and architect a solid communication adapter layer to decouple upper-layer business code from the underlying communication protocol.

ProtocolFeatures

MQTTTop choice for IoT, publish/subscribe model, lightweight

3. How to design device reconnection?

java // Exponential backoff reconnection strategy int retryCount = 0; int maxRetries = 10; while (retryCount < maxRetries) { try { connect(); break; } catch (Exception e) { retryCount++; long delay = Math.min(1000L * (1 << retryCount), 60000L); Thread.sleep(delay); // 1s, 2s, 4s, 8s... max 60s } }

4. What is the difference between MQTT QoS 0, 1, and 2?

QoSMeaningReliabilityPerformance
0At most onceMessages may be lostHighest
1At least onceMessages may be duplicatedMedium
2Exactly onceNo loss, no duplicationLowest

IoT scenarios typically use QoS 1, control commands use QoS 2.

5. How to design a high-availability device management platform?

  1. Multi-instance deployment: Cluster backend services with Nginx load balancing
  2. Independent communication service: Deploy device communication layer independently, decoupled from business services
  3. Redis caching: Store device real-time data in Redis to reduce database pressure
  4. Time-series database: Store historical data in TDengine / InfluxDB
  5. Message queue: Process device data through Kafka asynchronously for peak shaving
