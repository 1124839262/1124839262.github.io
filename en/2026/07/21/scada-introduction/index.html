1. Preface

Recently, an interviewer asked me, “Do you know SCADA?” My first reaction was: No, never touched it.

But thinking back, I’ve actually used PLCs (Siemens), KepServer (OPC Server), and explored JetLinks (IoT platform). Put these together, and aren’t they just the individual parts of a SCADA system?

This article is my own learning notes on SCADA, from the perspective of someone who has “used the parts but never seen the whole car.”

2. Understanding SCADA in One Sentence

SCADA = Supervisory Control And Data Acquisition.

In plain terms:

SCADA is the “big screen monitoring system” in a factory — it collects data from all on-site PLCs, sensors, and instruments, displays them as graphs, tables, and alarm lights, allowing operators to see the entire plant from a single computer and even remotely control equipment.

3. What Your Existing Knowledge Maps To in SCADA

What You KnowRole in SCADADescription
PLC (Siemens)Field Control LayerDirectly controls motors, valves, sensors — the “hands and feet” of SCADA
KepServerCommunication Gateway / OPC ServerConverts proprietary protocols from different PLC brands into a unified standard (OPC) for upper layers
JetLinksIoT Platform LayerDevice access, data storage, rule engine — similar to SCADA’s backend

Connecting them together:

PLC (Siemens) ──proprietary protocol──► KepServer ──OPC──► SCADA Host │ └──MQTT──► JetLinks (Cloud Platform)

You’ll notice: SCADA is not a single product, but a complete system architecture of “collection → transmission → display → control.”

4. SCADA System Architecture

A typical SCADA system has four layers:

┌────────────────────────────────────────────────┐ │ Layer 4: Management │ │ ERP, MES, Enterprise Systems, Cloud (JetLinks)│ │ Statistics, reporting, production scheduling │ └──────────────────────┬─────────────────────────┘ │ HTTP / MQTT / Database Sync ┌──────────────────────▼─────────────────────────┐ │ Layer 3: Supervision (SCADA Host) │ │ WinCC / IFIX / KingView / Ignition / ForceControl│ │ Real-time screens, trends, alarms, controls, │ │ history data │ └──────────────────────┬─────────────────────────┘ │ OPC / Modbus / Proprietary ┌──────────────────────▼─────────────────────────┐ │ Layer 2: Communication (Gateway / OPC) │ │ KepServer / OPC Router / Industrial Gateway │ │ Protocol conversion: Siemens S7 → OPC → Modbus │ └──────────────────────┬─────────────────────────┘ │ Profinet / Profibus / RS485 ┌──────────────────────▼─────────────────────────┐ │ Layer 1: Field (Device Layer) │ │ PLC, RTU, sensors, VFDs, instruments, valves │ │ Direct interaction with the physical world │ └────────────────────────────────────────────────┘

What You Know vs. What You Don’t

LayerYou KnowYou Don’t Know Yet
Layer 4JetLinksERP / MES
Layer 3❌ Complete blankWinCC / IFIX / KingView
Layer 2KepServer ✅Industrial Gateways
Layer 1Siemens PLC ✅RTU / VFDs

So when the interviewer asked “Do you know SCADA?”, they were most likely asking about Layer 3 — those configuration software packages.

5. Overview of Mainstream SCADA Software

SoftwareVendorFeaturesUse Case

KingViewWellinTechDomestic, cost-effective, easy to useSmall to medium projects

IntouchAVEVAVeteran, high market shareLarge projects

Selection Tips

  • If the client is all Siemens equipment → WinCC (ecosystem lock-in, hassle-free)
  • Budget-constrained, small to medium project → KingView / ForceControl (domestic, good enough)
  • Want modern web technology → Ignition (Java stack, friendly for you)
  • Power/petrochemical client → follow the client’s preference, IFIX or ForceControl

6. Core SCADA Functions

6.1 Real-Time Monitoring (HMI — Human-Machine Interface)

Draw the factory as a diagram, animate equipment status:

┌─────────────────────────────────────────┐ │ ┌─────┐ ┌─────┐ ┌─────┐ │ │ │ Pump1│ │ Pump2│ │ Pump3│ │ │ │ 🟢Running│ │ 🔴Stopped│ │ 🟡Fault│ │ │ └─────┘ └─────┘ └─────┘ │ │ │ │ Temp: 23.5°C Pressure: 1.2 MPa │ │ Flow: 156 m³/h Level: 2.8 m │ └─────────────────────────────────────────┘

6.2 Alarm Management

  • Temperature exceeds limit → popup + sound + SMS notification
  • Device offline → red flashing
  • Alarm classification: Critical / Important / General
  • Alarm acknowledgment: must be manually confirmed to clear

6.3 Trend Curves

  • Real-time trend: past 1 hour, 1-second refresh rate
  • Historical trend: past 3 months, for analyzing equipment degradation

6.4 Data Storage

  • Critical data stored every second
  • Historical data stored in relational DB (SQL Server / MySQL) or time-series DB (InfluxDB / TDengine)
  • Report export: daily, monthly, annual

6.5 Remote Control

  • Click buttons to remotely start/stop equipment
  • Set PID parameters
  • Operation audit trail: who, when, what they did

7. SCADA vs. JetLinks

Since you’ve explored JetLinks, here’s a comparison:

DimensionSCADA HostJetLinks
PositioningIndustrial on-site monitoringIoT device management platform
DeploymentOn-premise factory serverCloud / private deployment
UIConfiguration screens (flowchart style)Web Dashboard
ProtocolsOPC / Modbus / ProfibusMQTT / CoAP / HTTP
Real-timeMillisecondsSeconds
Data VolumeThousands of pointsHundreds of thousands of devices
ControlDirect device controlCommand dispatch via rule engine

In short: SCADA manages one factory; JetLinks manages a fleet of devices. They can complement each other — SCADA handles on-site real-time control, JetLinks handles cloud data aggregation and analysis.

8. How to Learn SCADA from Scratch

Step 1: Understand Configuration Software

Download a trial version of KingView (or similar) and get familiar with:

  • Variable definition (map PLC addresses to variables)
  • Screen configuration (drag controls, bind variables, create animations)
  • Alarm configuration (set thresholds, trigger conditions)
  • Report templates

Step 2: Connect Your Existing Knowledge

Siemens PLC ──S7 Protocol──► KepServer ──OPC──► KingView │ └──MQTT──► JetLinks

You’re only missing “KingView” — just one piece. Once you connect OPC data to the configuration software, you’ve built a complete SCADA system.

Step 3: Learn Common Protocols

ProtocolPurposeYour Proficiency
Modbus RTU/TCPUniversal device protocolAlready know ✅
S7 (Siemens proprietary)Siemens PLCIndirectly via KepServer
OPC UAIndustrial connectivity standardKepServer is an OPC Server
MQTTIoT protocolAlready know ✅
Profibus / ProfinetSiemens fieldbusJust understand the concept

Step 4: Hands-On Project

  • Use KepServer to simulate a PLC device
  • Connect KingView to KepServer’s OPC interface
  • Draw a simple process flow diagram, bind variables
  • Configure alarms and trend curves

9. How to Answer in an Interview

If asked “Do you know SCADA?”, you can answer:

“I mainly work on the lower layers of SCADA — I’ve used Siemens PLCs and KepServer for protocol conversion, and I’ve explored JetLinks for device management platforms. I haven’t directly used SCADA configuration software like WinCC or KingView, but I understand the complete data flow: PLC → KepServer → OPC → Configuration Software. If I need to learn the configuration software, I can pick it up quickly.”

This answer is both honest and demonstrates your knowledge framework.

Interview Questions

1. What does SCADA stand for? What are its core functions?

SCADA = Supervisory Control And Data Acquisition.

Core functions include: real-time monitoring (HMI), alarm management, trend curves, data storage, remote control, report generation.

2. What are the layers of a SCADA system?

LayerNameContent
Layer 4ManagementERP, MES
Layer 3SupervisionSCADA host (WinCC, KingView, etc.)
Layer 2CommunicationOPC Server, industrial gateway (KepServer)
Layer 1FieldPLC, RTU, sensors, instruments

3. What is OPC? Why is it needed?

OPC (OLE for Process Control) is a standard interface for industrial communication. Different PLC brands have their own proprietary protocols (Siemens S7, Mitsubishi MC, Omron FINS, etc.). If every SCADA host had to adapt to each one, the workload would be enormous.

With an OPC Server (like KepServer), it handles converting all proprietary protocols into a unified OPC standard interface, so SCADA hosts only need to support OPC — one interface, all devices.

4. What’s the difference between KepServer and a SCADA host?

KepServer is communication middleware — it only handles protocol conversion and data forwarding, with no display. A SCADA host (like WinCC) is a complete HMI — responsible for screen display, alarms, trends, reports, etc.

Their relationship: KepServer collects data → provides it via OPC to the SCADA host → the host displays it.

5. What’s the difference between SCADA and DCS?

DimensionSCADADCS
Control RangeGeographically dispersed (pipelines, water plants)Geographically concentrated (chemical plant units)
Real-timeLower requirement (seconds)Higher requirement (milliseconds)
Control MethodPrimarily monitoring, some controlPrimarily control, extensive closed-loop regulation
Typical ScenariosWater treatment, power dispatch, oil/gas pipelinesRefining, chemical, pharmaceutical

6. What’s the difference between SCADA and an IoT platform (like JetLinks)?

DimensionSCADAIoT Platform
PositioningIndustrial on-site real-time monitoringMass device remote management
Real-timeMilliseconds to secondsSeconds is fine
ProtocolsOPC / Modbus / ProfibusMQTT / CoAP / HTTP
DeploymentOn-premise factoryCloud
Device CountHundreds to thousands of pointsHundreds of thousands+

They can complement each other: SCADA handles on-site real-time control, IoT platform handles cloud data aggregation.
