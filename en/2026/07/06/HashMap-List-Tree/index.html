1. HashMap Overview

HashMap is one of the most commonly used data structures in Java. It is based on a hash table implementation and provides O(1) average time complexity for put, get, and remove operations. Starting from JDK 1.8, HashMap introduced red-black tree optimization, significantly improving performance under severe hash collisions.

Core Characteristics

  • Not thread-safe: Use ConcurrentHashMap in multi-threaded environments
  • Allows null key and null values
  • No ordering guarantee: Iteration order may differ from insertion order
  • Dynamic resizing: Default load factor 0.75, auto-resizes when threshold is reached

4.3 Red-Black Tree Properties

A red-black tree is a self-balancing binary search tree that satisfies these rules:

  1. Every node is either red or black
  2. The root node is black
  3. Red nodes must have black children (no consecutive red nodes)
  4. Every path from a node to its descendant NIL nodes contains the same number of black nodes
  5. Leaf nodes (NIL) are black

9. Summary

Key Takeaways

  1. Data structure evolution: JDK 1.8 introduced red-black trees to solve performance issues with long linked lists
  2. Linked list to red-black tree conversion: Requires both list length ≥ 8 AND array length ≥ 64
  3. Tail insertion: Avoids the infinite loop problem during resizing
  4. Threshold design: The gap between 8 and 6 prevents frequent conversions

Common Interview Questions

QuestionKey Points
Why is HashMap not thread-safe?Data loss during resize, infinite loop (1.7), data overwrite
Why red-black tree instead of AVL?Red-black trees have better insert/delete efficiency; AVL has slightly better lookup but higher maintenance cost
Why is the load factor 0.75?Balance between time and space, derived from Poisson distribution
How does HashMap resize?2x expansion, recalculate indices, preserve order
